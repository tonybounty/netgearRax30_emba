Usage: lsmtd [options] [<device> ...]

List information about memory technology devices.

Options:
  -u, --si-units       Scale sizes by factors of 1000 instead of 1024
  -b, --bytes          Print sizes in bytes
  -i, --ascii          Use ascii characters only
  -l, --list           Use list output format (default)
  -n, --noheadings     Don't print a heading
  -r, --raw            Use raw output format
  -P, --pairs          Use key="value" output format
  -J, --json           Use JSON output format
  -o, --output <list>  Comma seperated list of columns to print
  -O, --output-all     Print all columns
  -x, --sort <column>  Sort output by <column>
  -m, --no-ubi         Do not display information about UBI devices/volumes

  -h, --help           Display this help text and exit
  -V, --version        Output version information and exit

Available columns (for --output, --sort):
     DEVICE  name of the device node
    MAJ:MIN  major:minor device number
       NAME  device name string
       TYPE  device type
       SIZE  size of the device
    EB-SIZE  erase block size
   EB-COUNT  number of erase blocks
     MIN-IO  minimum I/O size
   SUB-SIZE  subpage size
   OOB-SIZE  out of band data size
         RO  read-only device
  CORRUPTED  wheather an UBI volume is corrupted
    REGIONS  number of additional erase regions
         BB  wheather the MTD device may have bad eraseblocks
     MAX-EC  current highest erase counter value on UBI devices
       FREE  available bytes on an UBI device or volume
   FREE-LEB  available LEBs on an UBI device or volume
  BAD-COUNT  number of bad physical eraseblocks
   BAD-RSVD  number of reserved eraseblocks for bad block handling

For more details see lsmtd(8).
