Usage:

   airiq_app [-i [INTERFACE_NAME]] [-p [IPADDR]] [-h] [-help]

For single radio interface 
-v                  Enable verbose output
-print_events       Enable printing of event data
-i [INTERFACE_NAME] Specify interface name. The default
                    interface used is the first 11AC interface 
-d [DWELL TIME]     Set the dwell time to be used for user-controlled scanning
                    Units: Milliseconds. Default value: 1000 ms.
                    Acceptable values: 400 - 1000 ms
-c [CAPTURE_TIME]   Set the capture time to be used for user-controlled scanning
                    Units: Milliseconds. Default value: 0 ms.
                    Acceptable values: 0 - 50 ms
-int [CAPTURE_INTERVAL]  Set the FFT capture interval to be used for user-controlled scanning
                    Units: Microseconds. Default value: 100 us.
                    Acceptable values: 50 - 500 us
-b                  Used to force the radio interface to 2.4GHz scanning only. 
-a                  Used to force the radio interface to 5GHz scanning only. 
-ch                 Used to specify a single scan channel in the 5GHz band
-home               Used to specify a single scan channel on the home channel in the 5GHz band. 
-core               Used to specify the core to use for the scan
-phy_mode           Used to specify the PHY mode to use for the scan 4x4 or 3+1
-p [IPADDR]         Set the IP address of airiq_service server. The default
                    to 127.0.0.1 for local connection
-h or -help         Display the command line help text.

Examples:
1. Scan both 2.4GHz and 5GHz band on single radio
dwell time: default; capture time: default; capture interval: default
    <airiq_app -i eth2>

2. Scan only 2.4GHz band on single radio
dwell time: 1000ms; capture time: 10ms; capture interval: 200us
    <airiq_app -i eth2 -b -d 1000 -c 10 -int 200>

3. Scan only 5GHz band on single radio
dwell time: 1000ms; capture time: 10ms; capture interval: default
    <airiq_app -i eth2 -a -d 1000 -c 10>

4. Scan only 5GHz home channel on single radio
dwell time: 1000ms; capture time: 10ms; capture interval: default
    <airiq_app -i eth2 -d 1000 -c 10 -a -home>

5. Scan 5GHz channel 36 on single radio
dwell time: 1000ms; capture time: 10ms; capture interval: default
    <airiq_app -i eth2 -d 1000 -c 10 -a -ch 36>



For multiple radio interfaces 
-i [2.4GHZ_RADIO] -i [5GHZ_RADIO]               Specify interface name. The first radio
                                                specified is the 2.4GHz radio and the second is 5GHz radio 
-d [2.4GHZ_DWELL_TIME] -d [5GHZ_DWELL_TIME]     Set the dwell time to be used for each radio I/F
                                                Units: Milliseconds. Default value: 1000 ms.
-c [2.4GHZ_CAPTURE_TIME] -c [5GHZ_CAPTURE_TIME] Set the capture time to be used for each radio I/F
                                                Units: Milliseconds. Default value: 0 ms.
-int [2.4GHZ_CAPTURE_INTERVAL] -int [5GHZ_CAPTURE_INTERVAL] Set the capture time to be used for each radio I/F
                                                Units: Microseconds. Default value: 100 us.
-ch                                             Used to specify a single scan channel in the 5GHz band
-home                                           Used to scan ONLY the home channel in the 5GHz band. 
-p [IPADDR]                                     Set the IP address of airiq_service server. The default
                                                to 127.0.0.1 for local connection
-h or -help                                     Display the command line help text.

Examples:
1. Scan 2.4GHz on first radio and 5GHz on second radio
dwell time: default; capture time: default; capture interval: default
    <airiq_app -i eth2 -i eth3>

2. Scan 2.4GHz on first radio and 5GHz on second radio
dwell time: 1000ms (eth2) 5000ms (eth3); capture time: 10ms; capture interval 200us
    <airiq_app -i eth2 -d 1000 -c 10 -int 200 -i eth3 -d 5000 -c 10 -int 200>

3. Scan 2.4GHz on first radio and 5GHz home channel on second radio
dwell time: 1000ms (eth2) 1000ms (eth3); capture time: 10ms; capture interval: default

4. Scan 2.4GHs on first radio and 5GHz channel 36 on second radio

***********Help for LTE_U scanning********

Usage:

   airiq_app -lteu [-i [INTERFACE_NAME]]  [-h] [-help]

For single radio interface 
-i [INTERFACE_NAME] Specify interface name. The default
                    interface used is the first 11AC interface 
-d [DWELL TIME]     Set the dwell time to be used for LTE-U scan
                    Units: Milliseconds. Default value: 200 ms.
                    Acceptable values: >= 200ms
-c [CAPTURE_COUNT]  Set the capture count to be used for LTE-U scan
                    Default value: 200 
-int [CAPTURE_INTERVAL]  Set the FFT capture interval to be used for LTE-U scan
                    Units: Microseconds. Default value: 656 us.
                    Acceptable values: Min 656us to multiples of 328us
-lte_cfg [LTE CONFIG REG VALUES]  Set the config for the LTE_U detector
                    values reg0 reg1 reg2 reg3 reg4 reg5 reg6 reg7 
-ch                 Used to specify a single scan channel in the 5GHz band
-h or -help         Display the command line help text.

Examples: 
1. Run with default settings 
airiq_app -lteu -i eth1

2. Increase the capture count and dwell time per channel for higher detection probability 
airiq_app -lteu -i eth1 -d 300 -c 400

3. Set a specific channel instead of scanning the default list of channels 
airiq_app -lteu -i eth1 -ch 36
